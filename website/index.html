MEASUREMENT LOGIC

単体テストでは、
共存時の犯人が見えない。

拡張Bの条件付き寄与

CPU全ON − CPUBだけOFF = +750 ms

正ならBが重くしている。負ならBが広告やページ処理を減らし、総合的に軽くしている。

01

拡張をまとめて選ぶ

manifest.jsonのあるMV3拡張フォルダを複数追加。サイト候補はmanifestから自動生成されます。

02

条件を直列で測る

全OFF、全ON、AだけOFF……を同じシナリオで実行。Chrome同士を競合させない直列計測です。

03

寄与の大きい順に見る

CPU寄与、Long Tasks、ヒープ差を並べ、SIGNIFとNOISEで意味のある差だけを判断できます。

REPORT / SCENARIO 01

「重い」だけで終わらない。
関数と行まで降りる。

結果はアプリ内と自己完結HTMLで確認できます。content script、Service Worker、Offscreen Documentを分け、Self / Total timeと呼び出し関係を残します。

  • 拡張別のCPU寄与を降順表示
  • 中央値と標準偏差でブレを判定
  • Chrome DevTools用.cpuprofileも保存

MEASUREMENT PRINCIPLES

PAIR

同じ反復内で比較

全OFFと全ONを近い時間で測り、サイトやネットワークの変動を抑えます。

SERIAL

Chromeは直列実行

複数プロセスのCPU競合を避け、比較そのものを壊しません。

CLEAN

使い捨てプロファイル

普段使うChromeの設定や履歴には触れず、条件を揃えて計測します。
